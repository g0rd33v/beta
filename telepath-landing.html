telepath — coordination layer for five surfaces.

Think once. Broadcast to five places. Native to each.

Today you ship to a website, then re-write for a channel, then re-script for a bot, then bolt on an app. telepath collapses that. One source, five destinations, native to each surface's grammar — web, PWA, Telegram Mini-App, channel, bot.

// what telepath changes

You shouldn't write the same idea five times.

A modern audience doesn't live in one place. They check Telegram before email. They install the app, then read the article on the laptop. They forward a channel post that took ten minutes to format. Re-shipping the same content is not a content problem. It's a coordination problem.

// shipping today

  • write the post once, reformat four times
  • each surface drifts on its own clock
  • edit on the site, channel goes stale
  • bot menu out of sync with web copy
  • analytics scattered across four dashboards
  • PWA and Mini-App disagree on "current"
  • no atomic publish, no rollback

// shipping with telepath

  • one source, five surfaces, one push
  • atomic broadcast — all or none
  • edit once, every surface converges
  • idempotent updates — no double-posts
  • signals from every surface in one stream
  • per-surface adapters, native dialect each
  • git-grade history per source artifact
// how it actually works

An event that arrives once and lands everywhere.

A source emits a structured event. telepath normalizes it, picks the right adapter for each surface, and broadcasts atomically. If a surface fails, retry. If you edit the source, every surface converges to the new state.

STAGE 01
»source

An event arrives

An article, a release, a status update, a Drafts commit. Anything that emits a structured event.

STAGE 02
»normalize

Reduce to one shape

Title, body, media, links, audience tags. A surface-agnostic canonical form the runtime can route from.

STAGE 03
»adapt

Five dialects, one event

Semantic HTML for web. OG card for share. Markdown-V2 for channel. Mini-App component. Inline keyboard for bot.

STAGE 04
»broadcast

Atomic fan-out

Push to all five. Retry on failure. Reconcile on edit. Last write wins, deterministically. No drift.

// the minimum config, in one file

One source. Five surfaces. No platform.

The config below is the entire integration. telepath.config.js goes at the bottom of any project that already has a source. Drafts emits the event. telepath broadcasts.

telepath.config.js// runtime · v0.1
import { telepath } from 'telepath'

export default telepath({

  // one source of truth
  source: {
    type:    'drafts',
    project: 'telepath',
    webhook: 'https://hooks.labs.vc/drafts'
  },

  // five surfaces
  surfaces: {
    web:      { domain:   'beta.labs.vc' },
    pwa:      { manifest: '/app.webmanifest' },
    miniapp:  { handle:   '@LabsRobot', route: '/app' },
    channel:  { id:       '@LabsVC', parse: 'MarkdownV2' },
    bot:      { handle:   '@LabsRobot' }
  },

  // optional review gate · retry policy
  review:  true,
  retries: 3
})
